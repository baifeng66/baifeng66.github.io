数据库

数据库 ( DataBase , 简称DB )

概念 : 长期存放在计算机内,有组织,可共享的大量数据的集合,是一个数据 “仓库”

作用 : 保存,并能安全管理数据(如:增删改查等),减少冗余…

服务启动

最后特别注意启动方式的问题：

运行brew service start mysql 可以后台启动mysql

运行mysql.server start 前台启动mysql

常用命令(不区分大小写)

表

数据库当中是以表格的形式表示数据。

任何一张表格都有行和列：

  • 行（row）:被称为数据/记录
  • 列（column）:被称为字段

每个字段都有字段名、数据类型、约束等属性

SQL分类

DQL:

​ 数据查询语言：

  • select

DML:

​ 数据操作语言（凡是对表中的数据进行增删改的都是）：

- insert
- delete
- update

主要操作表中的数据。

DDL：

​ 数据定义语言（凡是带有create、drop、alter的都是）

  • create
  • drop
  • alter

主要操作表中的结构。

TCL:

​ 事务控制语言

​ 包括：

​ 事务提交：commit

​ 事务回滚：rollback

DCL:

​ 数据控制语言

​ 例如：授权：grant、撤销权限：revoke

查询多个字段，用逗号隔开

起别名

利用 as 关键字。（只是显示将查询结果列名显示as后面的名字，原列表名并没有修改）

as 也可以省略。和别名之间加空格即可。

别名之间有空格或者是中文用 ‘ ‘ 扣起来即可。

列参与运算

条件查询

利用where关键字查询符合条件的数据。

下面是一些常见的条件查询操作符：

  • 等于：=。例如，column_name = value。

  • 不等于：<> 或 !=。例如，column_name <> value 或 column_name != value。

  • 大于：>。例如，column_name > value。

  • 小于：<。例如，column_name < value。

  • 大于等于：>=。例如，column_name >= value。

  • 小于等于：<=或者between…and…。例如，column_name <= value。

    • 当使用between…and…时，必须遵循左小右大
    • 而且其为闭区间，包括2端的值

  • is null: 判断字段是否为空（is not null 不为空）

  • and 并且：条件1 and 条件2 (条件要同时成立)

  • or 或者： 条件1 or 条件2 （哪个条件成立就查询）

  • and or同时出现，and 优先级比较高，如果想让 or 执行，得加小括号！

  • in 包含，相当于多个 or（not in 不在这个范围）

    • in 后面是具体的值

  • 模糊匹配：LIKE。例如，column_name LIKE pattern。可以使用通配符 % 表示任意字符，一个下划线_只匹配一个字符。

    • 要是想找名字里面有“_” ，要加转义字符 \
