与类的区别

结构（Struct）和类（Class）是两种不同的类型，它们有一些重要的区别。下面是结构和类之间的一些主要区别：

  1. 值类型 vs 引用类型：结构是值类型，而类是引用类型。当你将一个结构赋值给另一个变量或传递给方法时，会进行值的复制。而类的实例是通过引用进行传递和共享的。
  2. 内存分配：结构的实例在栈上分配内存，而类的实例在堆上分配内存。这意味着结构的创建和销毁速度更快，而类的实例需要进行垃圾回收。
  3. 默认构造函数：结构可以定义默认构造函数，它会自动初始化结构的字段。而类必须显式定义构造函数，否则会有一个默认的无参构造函数。
  4. 继承：类支持继承，可以通过派生类扩展和重用代码。而结构不能继承其他结构或类，但可以实现接口。
  5. 可空性：结构可以是可空的，即可以使用Nullable<T>类型来表示可能为空的结构。而类默认情况下是可空的，可以为其引用变量赋值为null。Nullable<Point> point = null;
  6. 性能：由于结构是值类型，它们通常比类更高效。结构的复制操作比引用传递更快，适用于存储小型数据和简单值类型。

结构适用于表示简单的值类型，如坐标、日期、颜色等。它们通常用于堆栈分配的小型对象，或者在需要高性能和低内存消耗的情况下使用。

类适用于表示复杂的对象，具有更多的功能和行为。它们通常用于堆分配的大型对象，或者在需要继承、多态和动态扩展的情况下使用。
需要根据具体的需求和场景来选择使用结构还是类。

在设计和编写代码时，理解结构和类之间的区别非常重要，以便选择适当的类型来满足需求。

重载

方法重载是指在同一个类中定义多个具有相同名称但参数列表不同的方法。通过方法重载，可以根据不同的参数类型和数量来调用不同的方法。

需要注意的是，方法重载的条件是方法名称相同，但参数列表必须不同（包括参数类型、参数个数或参数顺序）。

out

在C#中，使用out关键字可以将参数声明为输出参数。以下是一些使用out参数时需要注意的事项：

  1. 在方法内部，out参数必须在方法体内被赋值。这是因为out参数在方法调用之前不需要初始化，但在方法结束时必须有一个值。
  2. 在方法调用时，out参数必须在参数列表中使用out关键字进行声明。这样可以告诉编译器该参数是一个输出参数。
  3. 在方法调用之前，不需要对out参数进行初始化。因为在方法内部，out参数必须在方法体内被赋值。
  4. out参数可以在方法内部被赋值多次。这是因为out参数在方法调用之前不需要初始化，而是在方法内部被赋值。
  5. 在方法调用之后，可以使用out参数的值。因为out参数在方法结束时必须有一个值，所以可以在方法调用之后使用该值。
  6. 在方法调用时，可以使用out参数的引用传递。这意味着在方法内部对out参数的修改将反映在方法调用者的变量中。

总之，out参数是一种用于从方法中返回多个值的方式。使用out参数时，请确保在方法内部对其进行赋值，并在方法调用之后使用其值。

ref

在C#中，ref关键字用于传递参数的引用。通过使用ref关键字，可以在方法内部修改传递给方法的参数的值，并且这些修改将在方法调用结束后保持有效。

使用ref关键字时，需要注意以下几点：

  1. 在方法定义和方法调用的参数列表中都要使用ref关键字。
  2. 调用方法时，传递给ref参数的变量必须已经初始化。
  3. ref参数必须在方法内部进行初始化，否则会导致编译错误。
  4. ref参数可以是值类型或引用类型。

params

只限于数组

在C#中，params关键字用于定义一个方法参数，该参数可以接受可变数量的参数。使用params关键字可以使方法在调用时可以传递不同数量的参数，而不需要显式地创建一个数组来传递参数。

使用params关键字的方法可以接受零个或多个参数，也可以与其他参数一起使用。但是，每个方法只能有一个params参数，并且必须是方法的最后一个参数。

面向对象

面向过程是一种以过程为中心的编程方法，它将程序分解为一系列的步骤或函数，每个步骤按照特定的顺序执行。在面向过程编程中，数据和函数是分离的，函数通过传递参数来操作数据。这种编程范式适合解决简单的问题，但对于复杂的程序来说，代码可读性和可维护性可能会变差。

面向对象是一种以对象为中心的编程方法，它将程序组织为一组相互作用的对象。每个对象都有自己的状态（属性）和行为（方法），并且可以通过消息传递来与其他对象进行交互。面向对象编程强调代码的重用性和模块化，使得程序更易于理解、扩展和维护。

类

在C#中，类是一种用户自定义的数据类型，用于定义对象的属性和行为。对象是类的实例，它具有类定义的属性和行为。

属性

属性（Property）是一种特殊的成员，用于封装类的字段（Field）。属性提供了一种访问和修改类的字段的方式，同时还可以在访问和修改字段时执行额外的逻辑。

属性由两个访问器组成：get 访问器和 set 访问器。get 访问器用于获取属性的值，而 set 访问器用于设置属性的值。属性可以具有不同的访问修饰符，例如 public、private、protected 等。

权限修饰符

在C#中，权限修饰符用于控制类、方法、属性和字段的访问级别。

以下是C#中常用的权限修饰符：

  1. public：公共权限修饰符，表示可以从任何地方访问该成员。
  2. private：私有权限修饰符，表示只能在当前类中访问该成员。
  3. protected：受保护权限修饰符，表示只能在当前类及其派生类中访问该成员。
  4. internal：内部权限修饰符，表示只能在当前程序集中访问该成员。
  5. protected internal：受保护内部权限修饰符，表示只能在当前程序集中或者当前类及其派生类中访问该成员。
  6. 能够修饰类的访问修饰符只有2个：public,internal
  7. 子类的访问权限不能大于父类的访问权限，否则会暴露父类的成员

这些权限修饰符可以用于类的成员（方法、属性和字段），以控制它们的可见性和访问级别。根据需要，你可以选择适当的权限修饰符来限制或公开你的代码中的成员。

静态和非静态区别

  1. 非静态类中，可以有实例成员，也可以有静态成员
  2. 在调用实例成员的时候，需要使用 对象名.实例成员
    • 在调用静态成员的时候，需要使用 类名.静态成员

总结：

  • 静态成员必须使用类名调用，而实例成员使用对象名调用

  • 静态函数中，只能访问静态成员，不能访问实例成员

  • 实例函数中，可以访问静态成员，也可以访问实例成员

  • 静态类只允许有静态成员，不允许出现实例成员

string

在C#中，字符串（string）是一种表示文本数据的数据类型。以下是一些C#字符串的特性：

  1. 不可变性：字符串在创建后是不可变的，这意味着一旦创建，就不能更改字符串的内容。如果对字符串进行修改，实际上是创建了一个新的字符串对象。
  2. 字符串字面量：C#中的字符串可以使用双引号（”）括起来表示，例如：”Hello, World!”。这些被称为字符串字面量，它们可以直接在代码中使用。
  3. 字符串连接：可以使用加号（+）运算符将两个字符串连接起来，形成一个新的字符串。例如，”Hello, “ + “World!” 将得到 “Hello, World!”。
  4. 字符串插值：C# 6.0及更高版本支持字符串插值，可以在字符串中嵌入表达式或变量。使用$符号和花括号（{}）将表达式或变量括起来。例如，string name = "Alice"; string greeting = $"Hello, {name}!"; 将得到 "Hello, Alice!"。
  5. 字符串长度：可以使用Length属性获取字符串的长度，即字符串中字符的数量。
  6. 字符串比较：可以使用Equals方法或==运算符来比较两个字符串是否相等。注意，字符串比较是区分大小写的。
  7. 字符串格式化：可以使用字符串的Format方法或者使用$符号进行字符串格式化。这允许将变量或表达式的值插入到字符串中的特定位置。

值传递和引用传递

值传递（Pass by Value）是指将参数的值复制一份传递给方法或函数。在方法或函数内部对参数的修改不会影响原始变量的值。值类型默认是值传递。

引用传递（Pass by Reference）是指将参数的引用传递给方法或函数，使得方法或函数可以直接修改原始变量的值。引用类型默认是引用传递。

继承

继承是一种面向对象编程的重要概念，它允许一个类（称为子类或派生类）从另一个类（称为父类或基类）继承属性和方法。子类可以继承父类的成员，并且可以添加自己的成员或修改继承的成员。

继承的好处是可以实现代码的重用和扩展。子类可以继承父类的属性和方法，从而避免重复编写相同的代码，但是子类不会继承父类的私有字段。同时，子类还可以添加自己的特定行为和属性，以满足特定的需求。
需要注意的是，C#中的类只支持单继承，即一个类只能继承自一个父类。但是，可以通过接口实现多重继承的效果。
同时，子类并没有继承父类的构造方法，只是子类会默认调用父类的无参构造方法，从而创建父类对象，让子类可以使用父类中的成员。
