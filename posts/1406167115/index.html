目录结构

Linux没有盘符这个概念，只有一个根目录 / , 所有文件都在它下面, 目录结构为一个树形结构!

路径描述方式

路径之间的层级关系，使用：/来表示

比如：/usr/local/hello.txt –> 找到usr/local下的hello.txt

注意：

  • 开头的 / 表示根目录
  • 后面的 / 表示层级关系

练习

命令

命令基础

  • 命令行：即 Linux 终端或者其他连接软件，是一种命令提示符页面。可以使用各种字符化命令对系统发出操作指令
  • 命令：即Linux 程序，一个命令就是一个Linux 程序。没有图形化界面，可以在命令行提供字符化的反馈

命令格式

  • command：命令本身
  • -options：[可选]命令的选项，可以通过选项控制命令的行为细节
  • parameter：[可选]命令的参数，多数用于命令的指向目标

示例：

  • ls -l /home/hlc ，ls是命令本身，-l 是选项，/home/hlc是参数
    • 意思就是以列表的形式，显示/home/hlc目录的内容
  • cp -r test1 test2，cp是命令本身，-r 是选项，test1,test2是参数
    • 意思就是复制文件夹test1 成为 test2

ls命令

作用：列出目录下的内容，语法细节如下：

  • -a -l -h 是可选选项
  • Linux路径 是此命令可选的参数

当不使用选项和参数，直接使用ls命令本体，表示：以平铺形式，列出当前工作目录下的内容

Home 目录 和 工作目录

Linux系统的命令行终端，在启动的时候，默认会加载：

  • 当前登录用户的Home目录作为当前工作目录
  • Home目录：每个Linux操作用户在Linux系统的个人账号目录，路径在：/home/用户名
    • 如：图中的Linux用户是hlc，其Home目录是：/home/hlc

ls命令参数

  • ls 不使用参数，表示列出：当前工作目录下的内容
  • ls使用参数，表示：列出指定Linux路径的内容

通过 ls / 列出根目录下的内容：

ls命令选项

  • -a选项：表示：列出全部文件（包括隐藏的文件/文件夹）
  • 上图以.开头的，表示是Linux系统的隐藏文件/文件夹（只要以.开头，就能自动隐藏）
  • -l选项：表示：以列表的形式展示内容，并展示更多信息
  • -h 表示以易阅读的形式，列出文件大小，如K、M、G
    • -h选项必须搭配 -l 一起使用

ls命令选项的组合使用

写法：

  • ls -l -a -h
  • ls -lah
  • ls -alh

三种写法，效果一样，表示同时应用-l和-a的功能

ls命令选项和参数的组合使用

写法：

  • ls -la /

cd命令

作用：切换工作目录

语法：

  • cd命令无需选项，只有参数，表示切换到哪个目录下
  • cd命令不写参数直接执行，表示回到用户的 Home目录

pwd命令

作用：查看当前工作目录

语法：

  • pwd 无选项，无参数

练习

相对路径绝对路径和特殊路径符

相对路径和绝对路径

绝对路径：以根目录为起点，描述路径的一种写法，路径描述以 / 开头

相对路径：以当前目录为起点，描述路径的一种写法，路径无需描述以 / 开头

  • cd Desktop (相对路径写法)
  • cd /home/hlc/Desktop (绝对路径写法)

特殊路径符

  • . 表示当前目录
  • .. 表示上一级目录，cd ../.. 切换到上二级的目录
  • ~ 表示Home目录

练习

mkdir命令（只能创建文件夹）

通过 mkdir命令可以创建多级目录（文件夹）

语法：

  • -p: 可选参数，表示自动创建不存在的父级目录，适用于创建连续多层级的目录
  • 参数必填，表示Linux路径，即要创建的文件夹的路径（相对或绝对路径都可）

注意：

创建文件夹需要修改权限，目前请确保操作均在HOME目录下进行。

练习

touch-cat-more命令

touch命令(只能创建文件)

作用：创建文件

语法：

  • touch命令无选项，参数必填–>表示要创建的文件路径

cat命令

作用：查看文件内容

语法：

  • cat命令无选项，参数必填–>表示要查看的文件路径

more命令

作用：查看文件内容，但是支持翻页

  • cat命令是将内容全部显示出来
  • more命令支持翻页

语法：

  • more 命令无选项，参数必填–>表示要查看的文件路径

cp-mv-rm命令

cp命令

作用：复制文件/文件夹

语法：

  • -r 可选选项，用于复制文件夹的使用，表示递归
  • 参数1，linux路径，表示被复制的文件/文件夹
  • 参数2，linux路径，表示要复制去的地方

mv命令

作用：用于移动文件/文件夹

语法：

  • 参数1，linux路径，表示被移动的文件/文件夹
  • 参数2，linux路径，表示要移动去的地方，如果目标不存在，则进行改名，确保目标存在

rm命令

作用：删除文件/文件夹

语法：

  • 同cp命令，-r 选项表示删除文件夹
  • -f 选项表示强制删除（不会弹出提示确认消息）
  • 参数1 参数2 … 参数n 表示要删除的文件/文件夹，按照空格隔开

支持通配符 *，用来模糊查询

  • 符号 * 表示通配符，即匹配任意内容（包含空）
  • test*：表示匹配任何以 test 开头的内容
  • *test：表示匹配任何以 test 结尾的内容
  • *test*：表示匹配任何包含 test 的内容

演示：

  • 删除所有以test开头的内容
  • 演示强制删除
    • 通过su -root，并输入密码 123456 临时切换 root用户体验
    • 通过exit命令，退回普通用户

which-find命令

which命令

作用：查找命令的程序文件

我们在前面学习的Linux命令，其实它们本体就是一个个二进制可执行程序。

语法： which 要查找的命令

find命令

作用：查找指定文件

根据名字查找文件

语法：find 起始路径 -name "被查找文件名"

支持通配符 *，用来模糊查找

  • 符号 * 表示通配符，即匹配任意内容（包含空）
  • test*：表示匹配任何以 test 开头的内容
  • *test：表示匹配任何以 test 结尾的内容
  • *test*：表示匹配任何包含 test 的内容

根据文件大小查找文件

语法：find 起始路径 -size + | - n [KMG]

  • +、- 表示大于、小于
  • n 表示大小数字
  • KMG 表示大小单位，k-表示kb，M-表示MB，G-表示GB（k是小写，其他大写）

示例：

  • 查找小于10kb的文件： find / -size -10k
  • 查找大于100mb的文件：find / -size +100M
  • 查找大于1GB的文件： find / -size +1G

练习

grep-wc-管道符

grep命令

作用：从文件中通过关键字过滤文件行

语法： grep [-n] 关键字 文件路径

  • 选项-n，可选，表示在结果中显示匹配的行的行号
  • 参数，关键字，必填，表示过滤的关键字，带有空格或其他特殊符号，建议使用””将关键字包围起来
  • 参数，文件路径，必填，表示要过滤内容的文件路径，可作为内容输入端口

wc命令

作用：统计文件行数、单词数量、字符/字节数

语法：wc [-c -m -l -w] 文件路径

  • 选项 -c 表示：统计 bytes数量
  • 选项 -m 表示：统计字符数量
  • 选项 -l 表示：统计 行数
  • 选项 -w 表示：统计 单词数量
  • 参数 文件路径 表示：被统计的文件，可作为内容输入端口

管道符

作用：将管道符左边命令的结果，作为右边命令的输入

语法：命令1 | 命令2

echo-tail-重定向符

echo命令

作用：在命令行内输出指定内容

语法：echo 输出的内容

  • 无需选项，只有一个参数，表示要输出的内容，内容推荐使用 “ “包裹

`反引号

作用：被` 包裹的字符，会被当做命令执行

重定向符

` > 和 >>

  • ` > 表示：将左侧命令的结果，覆盖写入到符号右侧指定的文件中
  • ` >> 表示：将左侧命令的结果，追加写入到符号右侧指定的文件中

tail符

作用：查看文件尾部内容，跟踪文件的最新更改

语法： tail [-f -num] Linux路径

  • 参数 Linux路径 表示被跟踪的文件路径
  • 选项 -f 表示持续更新
  • 选项 -num 表示查看尾部多少行，默认10行

vi编辑器

三大模式

  • 命令模式
    • 命令模式下，所敲的按键编辑器都理解为命令，以命令驱动执行不同的功能
    • 此模式下，不能自由进行文本编辑
  • 输入模式
    • 也叫编辑模式、插入模式
    • 可以对文件内容进行自由编辑
  • 底线命令模式
    • 通过用于文件的保存、退出

Linux用户和权限

认知root用户

拥有最大的系统操作权限，而普通用户在许多地方的权限是受限的！

  • 普通用户的权限，一般在其HOME目录内是不受限的
  • 一旦出了HOME目录，大多数地方，普通用户仅有只读和执行权限，无修改权限

su和exit命令

su作用：切换账号

语法： su [-] [用户名]

  • `- 符号是可选的，表示是否在切换用户后加载环境变量，建议带上
  • 参数：用户名，表示要切换的用户，可省略 默认为root
  • 切换用户后，可通过 exit 命令退回上一个用户,也可使用快捷键 ctrl+d

sudo命令

临时为普通的命令授权，临时以root身份执行

语法：sudo 其他命令

为普通用户配置sudo认证

  • 切换到root用户，执行visudo命令，会自动通过Vi编辑器打开：/etc/sudoers
  • 在文件最后添加
  • 最后通过wq保存
  • 切换回普通用户
