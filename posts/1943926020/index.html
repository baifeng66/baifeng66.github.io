MVC 项目结构

  1. 控制器由Controller类实现，视图一般是扩展名为cshtml的文件,而模型则是自有属性的普通C#类。
  2. 控制器类的名字一般以Controller结尾，并且放到Controllers文件夹下。控制器的名字为控制器的类名去掉Controller。
  3. 视图一般放到Views文件夹下的控制器名的文件夹下。
  4. 视图→浏览器端提交的请求→模型→控制器→处理→模型→视图。渲染：Render。

ControllerBase Controller

2者的关系：Controller继承于ControllerBase,拥有其所有成员和功能。还提供了更多的MVC支持。

也可以建立一个普通的类，不需要继承它们，写上注解即可。


但是用不了 ControllerBase的成员和功能。

action方法-返回值

Action方法可以同步也可以异步，异步一般不需以Async结尾

web api中action方法的返回值如果是普通类型，那么返回值就会默认被序列化为Json格式

web api 中 action 方法的返回值同样支持 IActionResult类型，不包括类型信息，因此 Swagger等无法推测类型，所以推荐使用 ActionResult<T> ，其支持类型转换

捕捉URL占位符

  1. 在[HttpGet],[HttpPost]等中使用占位符，比如：{id},会从路径捕捉内容，从而供 action 方法的参数使用
  2. 捕捉的值会被自动赋值给Action中同名的参数，但是名字不一致，可以用[FromRoute(Name=”名字”)]
