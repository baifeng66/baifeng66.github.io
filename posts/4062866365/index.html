.Net Standard

是一个用于定义 .NET 平台 API 的规范。它的作用是为不同的 .NET 平台（如 .NET Framework、.NET Core 和 Xamarin）提供一致的 API 表面，以便开发人员可以编写可在多个平台上运行的可移植代码。

  • 只是一个标准，不是实现
  • 可以被支持其版本的 .Net Core 或者 .Net Framework等引用

Nuget

NuGet是一个强大的工具，它简化了在C#项目中管理外部库和依赖项的过程。它提供了方便的包管理功能，使您能够轻松地添加、更新和管理您的项目所需的各种功能和组件。

简单来说，功能和 maven 差不多

异步编程

是一种在C#中处理并发操作的技术。它允许您在执行耗时的操作时，不会阻塞主线程，从而提高应用程序的性能和响应能力。

比如像下图，如果没有开启多线程或者异步编程，用户提交请求后，只能等web服务器向web程序发生请求，等web程序的反馈才能进行下一次请求。

当开启异步编程后，当user提交请求，web服务器只进行发送请求给web程序，不等待它的反馈。继续从user哪里得到请求，继续发送给web程序。

但是相当于对一个user提交请求得到反馈的速度并不会加快，只是对整个程序的并发提升。

异步方法

异步方法是一种特殊类型的方法，它使用async关键字进行标记，并返回一个Task或 Task<T> 对象，表示异步操作的结果。

  • 异步方法的返回值一般是 Task<T>,T是真正的返回值类型。惯例 ：异步方法名字一般以Async结尾。比如：WriteAllTestAsync();
  • 即使方法没有返回值，最好把返回值声明为非泛型的 Task
  • 调用泛型方法时，一般在方法前加上 await，这样拿到的返回值就是 Task<T>里面 所指定 T 类型
  • 异步方法具有“传染性”：一个方法中如果有 await 调用，那么这个方法必须修饰为 async

这么看，同步和异步没有什么区别嘛！还多了这么多代码，其实异步在复杂的项目中起到的作用更大些。比如现在只写入1条数据，那我们写入10000条数据呢？

会报错！！！因为你在循环写入数据的同时不能去读取它。解决方法：在读取方法前加上 await。

Linq

IEnumerable+扩展方法

IEnumerable 接口是 LINQ 查询的基础，它允许您在集合上执行各种查询操作。通过实现 IEnumerable 接口，您可以创建自定义的可枚举类型，并使用 LINQ 查询语法或方法来对其进行查询和操作。

IEnumerable可以用于表示可枚举的集合类型。它定义了一个方法 GetEnumerator()，该方法返回一个实现了 IEnumerator 接口的迭代器对象，用于遍历集合中的元素。

使用 ToList() 或者 ToArray() 可以转换为集合或者数组类型

扩展方法（Extension Methods）是一种特殊的静态方法，允许您向现有的类型添加新的方法，而无需修改原始类型的定义或创建子类。

扩展方法的定义需要满足以下条件：

  1. 它必须是一个静态方法。
  2. 它必须定义在一个静态类中。
  3. 第一个参数必须使用 this 关键字进行修饰，并指定要扩展的类型。

通过定义扩展方法，可以在不修改现有类的情况下，为该类添加新的行为或功能。这对于向第三方库或框架中的类添加自定义方法非常有用，或者在自己的代码中为常用类型添加便捷的操作。

委托(复习)

委托是可以指向方法的类型，调用委托变量时执行的就是变量执行的方法

.NET中定义了泛型委托Action(无返回值)和Func(有返回值)，所以一般不用自定义委托,Func(类型1，类型ß2…)中最后一个是返回值类型

lamda表达式(复习)

委托变量不仅 可以指向普通方法，还可以指向匿名方法

匿名方法可以写成lamda表达式,去掉 delegate
还可以省略参数的数据类型，因为编译会根据委托类型推断出参数的类型，用 => 引出方法体。

如果委托没有返回值，且方法体只有一行代码，可省略 {}

如果=>之后的方法体中只有一行代码，且方法有返回值，那么可以省略方法体的{}以及return

如果只有一个参数，参数的()可以省略。

匿名类型

匿名类型（Anonymous Types）是一种临时创建的类型，用于在编译时定义具有一组属性的对象，而无需显式声明类的结构。

通常用于临时存储和传递数据，特别是在 LINQ 查询中。它们提供了一种方便的方式来创建临时对象，而无需显式定义类的结构。

需要注意的是，匿名类型是只读的，即不能修改其属性的值。它们的属性和类型是在编译时确定的，因此无法在运行时进行更改。

扩展方法

扩展方法（Extension Methods）允许你向现有的类或接口添加新的方法，而无需修改它们的源代码。扩展方法可以在不继承或修改原始类型的情况下，为类型添加新的行为。

在上述示例中，我们创建了一个静态类 StringExtensions，其中包含一个扩展方法 IsPalindrome。该方法接受一个字符串作为参数，并检查该字符串是否是回文。通过在方法的第一个参数前加上 this 关键字，我们将该方法定义为 string 类型的扩展方法。

在其他地方的代码中，我们可以直接在字符串实例上调用 IsPalindrome 方法，就像它是字符串类型的原始方法一样。这样，我们可以轻松地使用扩展方法来扩展现有类型的功能。

需要注意的是，扩展方法必须定义在静态类中，并且必须是静态方法。它们也必须在命名空间中可见，以便在其他代码中使用。扩展方法只是一种语法糖，它们并不会实际修改原始类型，而是在编译时被转换为对应的静态方法调用。

”滚动策略“

archiveAboveSize:

  • 单个日志文件超过多少字节就把日志存档。可以避免单个日志文件太大

maxArchiveFiles：

  • 限制数量个数的存档文件，旧的会被删

ruler

  1. rules节点下可以添加多个logger，每个logger都有名字（name属性），name是通配符格式的。

  2. logger节点的minlevel属性和maxlevel属性，表示这个logger接受日志的最低级别和最高级别。

  3. 日志输出时，会从上往下匹配rules节点下所有的logger，若发现当前日志的分类名符合这个logger的name的通配符，就会把日志输出给这个logger。如果匹配多个logger，就把这条日志输出给多个logger。但是如果一个logger设置了final=”true”，那么如果匹配到这个logger，就不继续向下匹配其他logger了
