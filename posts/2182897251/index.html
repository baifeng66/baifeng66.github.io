Ado.Net

基于Mysql开发

ADO.NET 是一组用于访问和操作关系型数据库的功能和类库。它是.NET Framework的一部分，提供了一种在C#应用程序中与数据库进行交互的方式。

以下是一些C# ADO.NET的主要功能：

  1. 数据库连接（Database Connection）：ADO.NET 提供了用于建立与数据库的连接的类，如 SqlConnection（用于连接 SQL Server）、OracleConnection（用于连接 Oracle 数据库）等。通过这些类，你可以建立与数据库的连接，并管理连接的打开和关闭。
  2. 数据库命令（Database Command）：ADO.NET 提供了用于执行 SQL 命令的类，如 SqlCommand、OracleCommand 等。通过这些类，你可以执行查询（SELECT）、插入（INSERT）、更新（UPDATE）和删除（DELETE）等数据库操作。
  3. 数据读取（Data Reading）：ADO.NET 提供了用于读取数据库查询结果的类，如 SqlDataReader、OracleDataReader 等。通过这些类，你可以逐行读取查询结果，并获取每行中的列值。
  4. 数据适配器（Data Adapter）：ADO.NET 提供了用于填充数据集（DataSet）和更新数据库的类，如 SqlDataAdapter、OracleDataAdapter 等。通过这些类，你可以将查询结果填充到数据集中，并将数据集中的更改反映到数据库中。
  5. 事务处理（Transaction Handling）：ADO.NET 提供了用于处理数据库事务的类，如 SqlTransaction、OracleTransaction 等。通过这些类，你可以在多个数据库操作中实现事务的管理，确保数据的一致性和完整性。
  6. 参数化查询（Parameterized Query）：ADO.NET 提供了用于执行参数化查询的功能，以防止 SQL 注入攻击。通过参数化查询，你可以将用户提供的输入值作为参数传递给 SQL 命令，而不是将其直接拼接到 SQL 语句中。

数据库访问模式

在C#中，有两种常见的数据库访问模式：连接模式和断开模式。

  1. 连接模式（Connected Mode）：
  • 连接模式是指在与数据库进行交互时，保持持久的数据库连接。在连接模式下，您需要显式地打开连接、执行查询或更新操作，并在完成后关闭连接。
  • 连接模式使用SqlConnection和SqlCommand等对象来执行数据库操作。您可以使用SqlDataReader来逐行读取查询结果，或使用ExecuteNonQuery方法执行非查询操作（如插入、更新或删除）。
  • 连接模式适用于需要实时访问数据库并进行复杂操作的场景。它提供了更直接的控制和更高的性能，但需要手动管理连接和事务。
  1. 断开模式（Disconnected Mode）：
  • 断开模式是指在与数据库进行交互时，临时连接到数据库，检索数据并断开连接。在断开模式下，您可以将数据存储在内存中的数据集（如DataSet）或数据表（如DataTable）中，然后在断开连接后进行操作。
  • 断开模式使用SqlDataAdapter来填充数据集或数据表，并提供了一些方便的方法来更新数据库中的更改。您可以使用数据集或数据表进行筛选、排序和修改等操作，然后使用SqlDataAdapter将更改应用到数据库中。
  • 断开模式适用于需要离线操作和对数据进行缓存的场景。它提供了更灵活的数据处理和更简化的代码，但在大量数据操作时可能会有性能损失。

MySqlDataReader

SqlDataReader类用于从SQL Server数据库中检索数据。它提供了一个只向前、只读的数据流，用于从SQL Server数据库中读取数据。SqlDataReader的主要用途是执行查询并逐行读取结果集。

MySqlDataAdapter

SqlDataAdapter是一个用于在断开模式下与数据库进行交互的类。它提供了一组方法和属性，用于填充数据集（DataSet）或数据表（DataTable），并将更改应用到数据库中。

以下是SqlDataAdapter的一些常见功能：

  1. 填充数据集或数据表：SqlDataAdapter的Fill方法可用于从数据库中检索数据，并将结果填充到数据集或数据表中。您可以指定要执行的SQL查询或存储过程，并提供一个数据集或数据表来存储结果。
  2. 更新数据库：SqlDataAdapter的Update方法可用于将对数据集或数据表所做的更改应用到数据库中。在调用Update方法之前，您需要设置适配器的InsertCommand、UpdateCommand和DeleteCommand属性，以指定相应的SQL语句或存储过程来执行插入、更新和删除操作。
  3. 自动生成命令：SqlDataAdapter可以根据数据集或数据表的结构自动生成插入、更新和删除命令。通过设置适配器的SelectCommand属性，您可以指定用于检索数据的SQL查询或存储过程。然后，通过调用SqlCommandBuilder的GetInsertCommand、GetUpdateCommand和GetDeleteCommand方法，适配器将自动生成相应的命令。
  4. 数据表映射：SqlDataAdapter提供了TableMappings属性，用于指定数据集中的数据表与数据库中的实际表之间的映射关系。这对于在数据集中使用不同的表名或列名来表示数据库结构非常有用。
  5. 参数化查询：SqlDataAdapter支持参数化查询，可以通过设置适配器的SelectCommand、InsertCommand、UpdateCommand和DeleteCommand的参数集合来传递参数值。这有助于防止SQL注入攻击，并提高查询的性能和安全性。

总之，SqlDataAdapter是一个强大的类，用于在断开模式下与数据库进行交互。它提供了方便的方法和属性，使您能够填充数据集或数据表，并将更改应用到数据库中。这使得您可以在离线环境中对数据进行操作，并将更改与数据库同步。
